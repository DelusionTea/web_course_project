# Многостраничный сайт МХАТ имени А.П. Чехова

Учебный/демонстрационный статический веб-сайт театра: четыре страницы, адаптивная вёрстка, загрузка контента через `fetch`, динамический репертуар из JSON, форма обратной связи и интеграция с Яндекс.Картами.

Подробное техническое задание см. в [PLAN.md](PLAN.md).

---

## Возможности

- **Главная** — hero-блок (фон задаётся в CSS), описание театра из текстового файла, HTML5-видео с постером, анонсы спектаклей из JSON.
- **Репертуар** — таблица цен (партер / амфитеатр / балкон) и карточки постановок, данные из `repertoire-items.json`.
- **О театре** — история и вводный текст из файлов, блок руководства, достижения, фотогалерея с модальным просмотром.
- **Контакты** — адрес и режим работы из файла, карта с меткой (координаты МХАТ), форма с клиентской валидацией.
- **Общее** — единый header и footer, подсветка активной страницы в меню, бургер-меню на узких экранах, индикатор загрузки при запросах, плавное появление блоков при прокрутке, иконка вкладки (`favicon.ico`).

## Требования

- Современный браузер с поддержкой ES6+, `fetch`, `IntersectionObserver`.
- **Локальный HTTP-сервер** — сайт рассчитан на загрузку `.txt` и `.json` через `fetch`; при открытии страниц как `file://` запросы к файлам часто блокируются политикой CORS.

## Быстрый старт

1. Клонируйте или скопируйте каталог проекта.
2. Запустите сервер из корня `Theater` (любой из вариантов).

**Python 3:**

```bash
cd Theater
python3 -m http.server 8765
```

Откройте в браузере: [http://localhost:8765/index.html](http://localhost:8765/index.html) (порт при необходимости замените на свой, например `8766`).

**Node.js (пакет `serve`):**

```bash
npx --yes serve Theater -l 8765
```

**Visual Studio Code / Cursor:** расширение «Live Server» — открыть `index.html` через «Open with Live Server».

## Редактирование контента

| Раздел сайта | Источник данных |
|--------------|-----------------|
| Краткое описание на главной | `resources/texts/main-description.txt` |
| Блок «О театре» на странице «О театре» | `resources/texts/about-theater.txt` |
| История | `resources/texts/history.txt` |
| Контакты в подвале и на странице контактов | `resources/texts/contacts.txt` |
| Таблица репертуара, карточки, анонсы на главной | `resources/texts/repertoire-items.json` |

Формат `repertoire-items.json`: массив объектов с полями `title`, `genre`, `duration`, `prices` (`parterre`, `amphitheater`, `balcony`), `description`, `image` (имя файла в `resources/images/`).

### Видео на главной

В [index.html](index.html) элемент `<video>` использует `videos/video.mp4` и постер `resources/images/female-mime-sitting-chair.jpg`. Чтобы добавить запасной источник (например, для браузеров без нужного кодека), можно указать несколько тегов `<source>` подряд.

Очень большие файлы (сотни МБ) при закрытии вкладки могут давать в логе сервера `BrokenPipeError` — это обрыв соединения клиентом, а не обязательно ошибка файла.

### Favicon

В корне лежит [favicon.ico](favicon.ico); во всех HTML-страницах подключено `<link rel="icon" href="favicon.ico" type="image/png" />`. Так браузер не запрашивает несуществующий `/favicon.ico` без ответа (файл при необходимости замените на свою иконку).

## JavaScript

### `js/main.js`

- Загрузка текстов и JSON, показ/скрытие `#page-loader` на время запросов.
- Построение таблицы и карточек репертуара, анонсов на главной.
- Валидация формы на `contacts.html` (email и обязательные поля), демонстрационный `alert` при успехе (отправки на сервер нет).
- Плавная прокрутка по якорным ссылкам, переключение мобильного меню, подсветка активного пункта навигации по `data-page` на `<body>`.
- Модальное окно галереи на `about.html`.
- Анимация появления элементов с классом `.reveal` через `IntersectionObserver`.

### `js/map.js`

- Инициализация карты после `ymaps.ready`.
- Центр и метка: **55.760837, 37.618423**, балун и подсказка: «МХАТ имени Чехова».
- При ошибке загрузки API показывается текст с подсказкой про ключ.

## Яндекс.Карты и API-ключ

В [contacts.html](contacts.html) подключён скрипт:

`https://api-maps.yandex.ru/2.1/?lang=ru_RU`

Для стабильной работы в продакшене Яндекс обычно требует **API-ключ** разработчика. Получить ключ: [Кабинет разработчика Яндекс](https://developer.tech.yandex.ru/). После получения подключите скрипт так:

```html
<script src="https://api-maps.yandex.ru/2.1/?apikey=ВАШ_КЛЮЧ&lang=ru_RU" defer></script>
```

Без ключа карта может не отображаться — это ограничение политики API, а не ошибка вёрстки.

## Стили и адаптивность

- **Цвета (CSS-переменные):** бордовый `#8B0000`, золотой `#D4AF37`, фон `#FFF8DC`, текст `#2C2C2C`.
- **Шрифты:** Playfair Display (заголовки), Roboto (текст), подключение через Google Fonts в HTML.
- **Сетка:** общий каркас на CSS Grid и Flexbox; контейнер до `1200px` (на ширине `1440px+` — до `1320px` в `responsive.css`).
- **Точки перелома:** `320px`, `768px`, `1024px`, `1440px` — см. [css/responsive.css](css/responsive.css).
- **Hero-секция на главной:** фоновое изображение задаётся в [css/style.css](css/style.css) (селектор `.section--hero::before`).
